# 건강이 - 고령층을 위한 병원 복지 서비스 자동 추천 챗봇

고령층이 자연어로 증상과 위치를 입력하면, 공공데이터와 연동하여 적합한 병원 및 복지기관을 추천하는 모바일 챗봇 애플리케이션입니다.

> 미래내일일경험 사업 (2025.05.20 ~ 2025.06.30) | 참여기관: 동구노인종합복지관

 주요 기능

① 자연어 입력 기반 증상 분석**
"무릎이 아파요"와 같은 자연어 문장에서 키워드를 추출하여 진료과와 연동합니다.

② 유사어 매칭**
soynlp 기반 유사어 사전을 활용해 다양한 표현을 처리합니다.
예: "말이 잘 안 나와요" → 신경과

③ 위치 기반 병원 추천**
사용자가 입력한 지역명(동 단위)을 기준으로 공공데이터에서 병원 정보를 필터링하여 JSON 형태로 추천합니다.

④ 고령층 친화형 챗봇 UI**
큰 글씨, 단순한 버튼, 다크/라이트 모드 전환, TTS 음성 안내 기능을 제공합니다.

⑤ 실시간 응답 출력**
병원명, 주소, 추천 진료과를 포함한 결과를 즉시 응답합니다.

---

 시스템 구조

```
사용자 입력 (자연어: 증상 + 지역)
    ↓
React Native + Expo (프론트엔드)
    ↓ JSON POST /recommend
FastAPI 서버 (백엔드)
    ├── 유사어 사전 매핑 (soynlp + CSV)
    ├── 증상 → 진료과 분류
    └── 지역 필터링 → 병원 추천
    ↓ JSON 응답
챗봇 화면 출력 (병원명 / 주소 / 진료과)
```

 팀장으로서 주요 기여

### 챗봇 설계 및 API 인터페이스 정의

- 프론트엔드(React Native)와 백엔드(FastAPI) 간 `/recommend` 엔드포인트의 요청/응답 구조를 사전 정의하여 병렬 개발을 가능하게 하였습니다.
- 입력 포맷 오류(쉼표 미사용 등) 발생 시 봇이 자동으로 안내 메시지를 출력하는 예외 처리 로직을 설계하였습니다.

### 기술적 문제 해결

- **모바일 백엔드 연결 오류**: Android/iOS 환경에서 로컬 주소 처리 방식 차이로 서버 응답이 오지 않는 문제를 ngrok 퍼블릭 터널링 도입과 `Platform.OS` 조건 분기 처리로 해결하였습니다.
- **TTS 중복 재생**: 챗봇 응답이 연속 수신될 때 음성이 겹쳐 재생되는 문제를 `stopTTS()` 호출로 제어하고, 화면 이동 시 자동 종료되도록 개선하였습니다.

### 프로젝트 매니지먼트

- 주 2~4회 정기 회의를 통해 팀원 간 진행 상황을 공유하고 일정을 관리하였습니다.
- 팀원 코드를 정기적으로 리뷰하고 GitHub 기반 최종 병합을 직접 관리하여 코드 품질을 유지하였습니다.

실행 방법

```
# 백엔드
cd backend
pip install -r requirements.txt
uvicorn main:app --reload

# 프론트엔드
cd frontend
npm install
npx expo start
```

> 모바일 실기기 테스트 시 ngrok으로 백엔드를 퍼블릭 터널링 후 프론트엔드 API URL을 변경하세요.
